# ZHI（栀）— Zero-Hypothesis Intelligence Ecosystem

## 我们在问什么

一个根本性的问题：

> 如果没有任何预设的社会规则，没有"合作"的指令，没有"语言"的定义，仅仅给定生存压力、有限资源、局部感知和行动能力——复杂行为能否从零开始涌现？

ZHI 是一个**最小数字生态系统**，试图用最简单的规则观察最复杂的结果。

---

## 设计哲学：不预设

系统中不存在友方、敌方、阵营、社会关系、语言含义、合作规则。

我们只定义七件事：

- **生存** — 统一能量货币（Energy）驱动一切：所有主动动作（移动/攻击/信号）直接从 Energy 扣除。Energy 仅能通过进食获取，无被动恢复。Energy 每 tick 衰减（基础 0.1/tick + 尸体污染 + 年龄衰老 + 低温代谢加速 + 脱水惩罚）。水分（Water）独立衰减（0.1/tick），高温加速失水，饮水恢复。当 Energy ≤ 0 时 Agent 死亡
- **资源** — 生态闭环驱动：植物拥有完整四阶段生命周期（Seed→Sprout→Adult→Decay）。种子休眠等待适宜条件（养分>0.4 + 地下水>0.25 + 光照 + 适宜温度），萌芽后快速生长但脆弱（环境容差窄），成熟后可随风散播种子（支付能量代价），衰老后进入腐烂阶段缓慢归还养分到土壤。各阶段可食性不同（Sprout 60%/Adult 100%/Decay 30%），Seed 不可食且不散发食物气味。Agent 进食植物获取能量，Agent 死亡后尸体腐烂返还养分。植物生长受温度钟形曲线（最适 25°C）、日照、地下水、养分影响，极端温度（<-2°C / >45°C）致死。尸体腐败 → 养分扩散 → 植物生长 → Agent 进食 → 死亡腐败，形成完全闭合的物质能量循环。**植物生命周期 v4.7**：四阶段状态机替代单态能量模型，Seed/Sprout/Adult/Decay 各有独立生长/存活/繁殖规则，环境条件触发阶段转换
- **空间** — 64×64 网格，移动有代价；每格最多同时站 2 个 Agent，超限则移动被阻止，形成自然空间竞争；随机河流贯穿地图（5 格宽，浅水可通行，深水不可跨越），河流带有随机浅滩（可穿越的渡口），每代重新生成
- **环境** — 24 小时昼夜循环（12 现实分钟 = 1 天）；温度正弦变化（5°C 黎明 ~ 35°C 午后）；Agent 拥有体温属性（惯性追踪环境温度），散发热量（自身格 +2°C × HP比例，周围 8 格 +1°C × HP比例），受伤时体热下降；河流冷却周围；低温（体温<15°C）触发额外 HP 衰减，抱团取暖可减缓体温下降；高温（体温>30°C）加速口渴；地图热力图叠加 + 每格温度 tooltip。**风与气压 v4.5**：温度差产生气压差（暖低冷高），气压梯度产生风（2D 向量场），风驱动温度平流（半拉格朗日平流）、气味远距离输送、种子方向性扩散、蒸发加速。Dashboard 可叠加气压热力图和风箭头。**动态水循环 v4.4**：随机降雨（湿度驱动强度，圆形 falloff），地表水沿高程梯度流动（陡坡径流加速 ×SlopeRunoffMult），河流作为水槽排水（地表水汇入河流后排出系统，河流自身加速排水），白昼蒸发（与温度、风速正相关），地表水↔地下水交换（土壤渗透率由地形推导：近河冲积层高渗透率，高海拔薄土低渗透率，陡坡低渗透率），地下水 4 邻域扩散。**旱/雨季节律**：~10000 tick sin 波驱动湿度振荡，干季温差扩大 1.3× 雨少，湿季温差缩小雨多植物繁盛。**生态区与光照 v4.6**：环境衍生 Biome 系统——综合 Height、Groundwater、SurfaceWater、Nutrient、Temperature、DistanceToRiver 六维变量自动分类为 8 种生态区（水域/河岸/荒漠/草原/丛林/湿地/高地/谷地），各生态区作为倍增修改器影响蒸发速率、Agent 体温变化速率、植物生长倍率。光照系统：太阳高度角 sin 曲线（正午峰值 × 坡向修正——南坡加成×1.5、北坡惩罚，北半球假设）× 高地额外光照。光照加热地面温度网格（SolarHeatingRate），加速水面蒸发（SunEvaporationMult），增强植物光合作用（SunPhotosynthesisBoost=2.0×）。Dashboard 提供生态区颜色叠加层（8 色），hover 显示生态区名称、温度、土壤参数
- **感知** — 7×7 方向性视野（agent 面朝方向可见锥形区域，背后不可见）；没有全局地图；水流声音传播 10 格（Manhattan 距离，逐格衰减 0.9）；气味和食物气味随风平流输送，形成远距离化学线索
- **需求** — 统一代谢系统：Energy（能量）和 Water（水分）两轴。Energy 每 tick 消耗（基础代谢 + 年龄衰老 + 尸体污染 + 低温代谢加速 + 脱水惩罚）。Water 每 tick 衰减（基础 + 高温加速）。Energy ≤ 0 死亡。代谢级联：冷→代谢加速→能量消耗增加→死亡；热→水分流失→脱水→能量惩罚
- **行动** — 8 个离散动作：上下左右移动（平地 0.5 Energy × BodySpeed倍率，浅水 +1.0，深水 +2.5，深水爬岸 +1.0）、进食（每 tick 提取食物能量，多 agent 可同时抢食同一食物）、攻击（冷却 4 tick，+0.5 Stress，消耗 8 Energy / BodyStrength，伤害 × BodyStrength倍率）、释放化学信息素（连续值 0-1，sigmoid-gaussian 采样，消耗 3 Energy，半径 4 Chebyshev 扩散至 ChemicalField，同时广播至附近 Agent 的 ChemicalMemory）、饮水（需靠近水源，+40 Water）

仅此而已。其余一切——如果出现了的话——都是涌现的。

化学信息素系统是最好的例子。Agent 可以释放连续强度 0-1 的化学信息素（sigmoid-gaussian 采样，消耗 3 Energy，低能量时阻止发射）。信息素以 Agent 自身为圆心呈波状扩散——半径 4（9×9 区域），Chebyshev 距离线性衰减，单次脉冲触发后通过 4 邻域扩散 + 0.95/tick 自然消退，形成持久的化学场。释放量受 BodyFat 加成（脂肪越多释放越强）。系统不规定信息素的语义——Agent 必须自己学习在何时、何地、释放多强的信息素。如果某种释放模式稳定出现了，那是因为它有生存优势，而不是程序员写了一个语义表。

## 为什么是"最小"

我们刻意把系统做得尽可能小：

只有**64 个 Agent**（种群上限，可配置），数量少意味着每个个体的生死都影响群体，资源竞争是真实的，不存在"大数定律下的平均存活"。

**340 维观察向量**，Agent 看到的是：

- [0-293] 7×7 方向性网格编码（49 格 × 6 通道：植物能量/地下水饱和度/尸体/其他 Agent/自身/地形），可见区域由 Agent 朝向决定（面朝方向锥形可见，背后不可见）。通过 CNN 提取空间特征（纯卷积，无 Pooling：6ch→Conv(k=5,p=0)→16@3×3→Conv(k=3,p=0)→32@1×1），压缩为 288 维特征向量
- [294] Energy/100, [295] Stress/5, [296] LastAction/7, [297] Age, [298] reserved
- [299] 化学记忆（单值连续衰减，每 tick ×0.95）
- [300-303] Agent 气味梯度（N/S/E/W 方向相邻格差值）
- [304-306] 局部统计（可见食物数/5，可见 Agent 数/8，当前格气味/10）
- [307-308] 朝向向量（facing_x, facing_y）
- [309] 化学新鲜度（ChemicalAge/20）
- [310] Water/100, [311] reserved, [312] 水流声音/10
- [313] 进食状态, [314] 静止状态
- [315-320] 身体参数（Size/Speed/Strength/VisionRange 映射 0.3-2.5→0-1；Fat/ColdResist 直接 0-1）
- [321] 高程（-10..+10 → 0-1）
- [322-325] 化学场梯度（N/S/E/W 方向相邻格差值）
- [326-339] 保留

**CNN + GRU 混合架构**：7×7×6 方向性网格先经两层纯卷积提取空间特征（6ch→Conv(k=5,p=0)→3×3×16→ReLU→Conv(k=3,p=0)→1×1×32→ReLU），展平为 288 维；再与其余 46 维非网格观测拼接为 334 维向量，送入 GRU(334→128)→FC(128→64)→Actor(8)/Chemical(1)/Critic(1) 三头。Chemical 输出经 sigmoid + 固定标准差 0.15 高斯采样产生连续释放强度。

**可遗传身体参数（Genome）**：每个 Agent 拥有 6 个可遗传参数——Size（体型 0.3-2.5，影响能量消耗速率）、Speed（速度 0.3-2.5，降低移动耗能）、Strength（力量 0.3-2.5，增加攻击伤害）、VisionRange（视野 0.3-2.5）、FatStorage（脂肪 0-1，增强信息素释放量）、ColdResistance（抗寒 0-1，减少低温伤害）。所有参数通过能量经济耦合：Size 越大每 tick Energy 消耗越快，Speed 越高移动越省能。繁殖时参数变异（高斯噪声 std=0.05）传递给后代，创造可遗传的生态位分化。

**地形与高程**：世界生成连续高程图（4 倍频值噪声，-10 至 +10）。河流沿高程梯度生成，自然流向低处。高程影响：坑内视野缩窄、攻击伤害降低但夜间保暖；高地上视野开阔、攻击伤害提升。高程作为 Agent 感知通道之一，为未来的"趋高/趋低"策略分化提供基础。

**8 个动作**，不是连续控制空间。上下左右、吃、攻击、释放化学信息素、饮水。复杂行为必须从这些原子操作的组合中涌现。移除了所有文明预设动作（推挤/推人/拉人/地形改造），将行为空间回归到最基本的生存操作。

**极简奖励**：只奖励存活和进食，不奖励攻击、不奖励合作、不奖励通信。攻击是否有价值，合作是否有利，信号是否有用——全靠 Agent 自己发现。

**统一代谢系统**：Energy 和 Water 两轴，Energy 是统一的能量货币——所有主动动作（移动/攻击/信号）直接从 Energy 扣除，无被动恢复，只有进食能获取。Water 独立衰减，高温加速失水，饮水恢复。代谢级联：冷→代谢加速→能量消耗增加→死亡；热→水分流失→脱水→能量惩罚。这确保 PPO 必须学习权衡移动、攻击、寻食、饮水的经济性。

**昼夜与温度**：24 小时昼夜循环（12 现实分钟 = 1 天 = 3600 tick）。温度正弦变化——黎明 04:00 最冷 5°C，午后 14:00 最热 35°C。

- **Agent 体热**：每个存活 agent 散发微量体温（自身格 +0.3°C，周围 8 格各 +0.15°C），热量按 HP 比例缩放。静止时额外微增（自身 +0.5°C，邻居 +0.3°C）。大量聚集时形成可察觉的微气候——10 个静止 agent 抱团核心格升温约 8°C。

- **热力物理 v4.3**：温度网格持久化（增量更新而非覆写），引入热扩散传导与水体高热容。水体目标温度比气温低 14°C（`WaterCoolingOffset=14`），确保河流常年凉爽；水体变温速率仅为陆地的 1/4（`WaterHeatCapacity=4.0`），产生热惯性；深水额外偏冷 3°C。热量通过 4 邻域热扩散传导（`ThermalDiffusionRate=0.12`），冷热沿河岸向陆地渗透。预计算距离至河流场（`DistanceToRiver`），陆地变温速率随距河流远近连续过渡（`RiverLandInfluence=8` 格范围指数衰减），形成自然的河流微气候梯度。**高程温度递减**：每格温度随高程线性变化（`HeightLapseRate=0.5`，即高地比低地冷 ~10°C），创造从暖谷到寒山的自然温度分层。**Agent 体温**：每个 Agent 拥有独立体温属性，以 0.05/tick 速率向环境温度渐变（水中 2x 加速：0.1/tick），硬下限 26°C。**失温系统**：体温低于 33°C 开始线性扣 Energy（26°C 时最高 0.08 Energy/tick，约 4 分钟致死窗口）。深水夜晚极端场景有效温度可低至 -3°C，体温快速降至 26°C 触发最大失温伤害——必须尽快上岸。白天高温（>30°C）加速 Water 衰减（最高 1.5x），推动 agent 靠近水源。Dashboard 地图提供温度热力图叠加（蓝色→绿色→橙色→红色），开启后每格 hover 显示精确温度值，可直观观察热岛效应与河流冷带。

- **河流与深水**：5 格宽的随机河流，中心 1 格深水（可通行但消耗大量能量），两侧浅水（可通行）。深水移动额外消耗 2.5 Energy（浅水 +1.0），从深水爬上岸额外 +1.0 Energy。能量耗尽则困于深水中。重生仍避免深水。随机出现浅滩（深水变浅水），成为天然渡口。

**进食切换**：Eat 动作不再是瞬时操作，而是切换进食状态。进入进食状态后每 tick 自动从食物中提取能量，执行任何非 Eat 动作（移动/攻击/信号/饮水）即退出。进食中的 agent 受到攻击时承受 110% 伤害（进食脆弱性），必须权衡安全与能量获取。

**食物竞争**：多个 agent 同时进食同一植物时，每 tick 提取效率按 1/√n 递减（如 2 人各得 71%，3 人各得 58%），创造抢食压力。

**尸体污染与动态腐烂**：尸体周围 2 格范围内 agent 遭受额外 HP 衰减（最高 0.06/tick 在尸体格），防止 agent 围绕尸体集群，推动探索。尸体腐烂速率由温度驱动（高温加速、寒冷减缓）和湿度驱动（雨季加速），大型尸体（>50 Energy）释放 2× 养分为土壤增肥，形成自然绿洲效应。

**攻击冷却**：每次攻击后 4 tick 内不可再次攻击，防止连续秒杀。攻击伤害受 Energy 比例缩放（Energy 越低伤害越低，Energy 越高伤害越高），鼓励健康管理而非无脑攻击。

**渐进衰老**：5000 tick 后 Agent 开始衰老（额外衰减 +0.02/tick），6000 tick 加速到 +0.05，7000 tick 达到 +0.1，8000 tick 自然死亡。这确保世代更替真实存在，防止超级个体垄断种群。

**能量优先（Energy-First）**：Agent 拥有 Energy（0-100）作为统一能量货币。所有主动动作（移动/攻击/信号）直接从 Energy 扣除。Energy 无被动恢复——只有进食能获取能量，食物链是唯一能量来源。攻击消耗 = AttackCostBase / BodyStrength（强者更节能）。连续 5 tick 不移动进入静止状态——受攻击伤害 ×1.2（易伤），自身格和周围格产热增加（+0.5°C/+0.3°C）。当 Energy 低于 LowEnergyThreshold（10）时移动有 50% 概率失败。

**地形物理（风化与水漫）**：Pit 和 Mound 拥有 2000 tick 生命周期（TTL），过期自动风化回平地。靠近河流的 Pit 每 tick 可能被水淹没变成动态浅水——一旦与河流连通即为永久水域（不再蒸发干涸），形成不可逆的环境改变。Mound（土包/堤坝）可阻挡水流蔓延。不再支持 Agent 主动地形改造——地形变化完全由自然物理驱动。

**可遗传身体参数（Genome）**：Agent 的身体属性（Size/Speed/Strength/VisionRange/FatStorage/ColdResistance）由 Genome 编码，在繁殖时通过高斯变异（std=0.05）传递给后代。各参数通过能量经济与生存耦合：Size 越大基础代谢越高（Energy 消耗更快），Speed 越高移动越省能（MoveCost 降低），Strength 越高攻击伤害越大且攻击耗能越低，FatStorage 越高化学信息素释放越强，ColdResistance 越高低温伤害越小。这套系统为生态位分化提供进化基础——Agent 不会"全能"，必须专精。

这种设计的核心假设是：**如果一个行为在最小系统中都无法涌现，那它在更大更复杂的系统中也不太可能是"真的"涌现，而更可能是人的偏见在起作用。**

## 技术与原理

底层是 PPO（近端策略优化）+ GAE（广义优势估计），Agent 的大脑是一个 CNN+GRU 混合网络。CNN 处理 7×7×6 方向性网格提取空间特征，GRU 融合空间特征与非网格状态进行时序决策。所有 Agent 共享一个 BatchBrain（GPU 批量推理），PPO 在线持续训练。每次 PPO 更新后清空 GRU 隐藏状态，确保策略更新后不携带旧策略的时序偏差。

世界**永不重置**——采用个体轮回机制：每个 Agent 死亡后经过 5 秒延迟（25 tick）自动重生，在新位置重新开始，继承共享 PPO 策略的经验（GRU 隐藏状态清零，但网络权重持续进化）。每 16 次轮回计为一代（Generation），每 8 代保存最佳权重至数据库。

遗传算法不再用于整代替换，而是用于繁殖：存活时间长、HP 高的 Agent 可以通过无性繁殖产生后代（权重变异），后代加入种群（受种群上限约束）。

技术栈很简单：.NET + TorchSharp 做推理与训练，React + Canvas 做可视化，WebSocket 推送实时状态。

**GPU 优先内存架构**（v4.3）：PPO 经验缓冲直接存储于 GPU 显存（预分配张量），每步增量写入而非大批量 PCIe 传输；StateMatrix 复用 GPU 张量避免每 tick 分配/释放；每 tick 临时数组池化复用减少 GC 压力。推理/训练全程在 GPU 上进行，世界模拟保留在 CPU。

## 观察与调试

实时仪表盘提供多个维度的观察：

**顶栏** — 显示世界时钟（☀/🌙 图标）、温度（彩色标注：白<5°C→蓝→绿→橙→红>35°C）、世代、死亡数、存活数、在线状态。DB 统计区显示平均寿命、夜间死亡率、人均攻击/进食/信号次数。

**事件监视器** — 结构化的世界事件流：进食、攻击、死亡、繁殖、复活、化学信息素释放、淹没、风化。可按事件类型过滤，支持清除和自动滚动。

**3D 视图** — 通过 "3D" 切换按钮可从 2D Canvas 切换到 Three.js 3D 渲染视图。3D 视图支持鼠标旋转/缩放/平移（左键旋转、中键缩放、右键平移），地形以高度位移呈现（HeightMap 0-255→5 单位高差），8 色生态区顶点着色，Agent/植物/尸体使用 InstancedMesh 实例化渲染（单次 draw call），水面半透明蓝色层，光照跟随昼夜循环（方位角+色温+强度变化）。

**显示切换** — 可以叠加多种可视化层（默认全部开启：气味-栀、气味-食、方向、视野）：

- **气味-栀**（紫色）— Agent 移动时留下的气味痕迹，每 tick 衰减和扩散。反映 Agent 活动热点区域
- **气味-食**（绿色）— 植物和尸体散发的气味，独立于 Agent 气味扩散。植物气味强度与能量成正比，帮助 Agent 定位高能量食物
- **朝向箭头** — Agent 的朝向方向，决定视野方向
- **视野区域** — 以 Agent 朝向为基础的 7×7 锥形可见区域（静止时缩小为 3×3 圆形）
- **化学场**（黄色）— 空间中残留的化学信息素痕迹（连续值），半径 4 圆形扩散 + 4 邻域扩散，中心亮边缘淡，自然衰减。反映通信热点
- **生态区**（8 色，默认关闭）— 水域(蓝)、河岸(浅绿)、荒漠(黄)、草原(绿)、丛林(深绿)、湿地(青)、高地(棕)、谷地(橄榄绿)
- **地形**（棕色/蓝色，默认关闭）— 坑（深褐色）、土包（浅褐色）、水坑（浅蓝），hover 显示 TTL 倒计时；TTL<200 时出现风化裂纹
- **水流**（天蓝色箭头，默认关闭）— 8 方向箭头显示河流流向，仅在缩放足够大时可见
- **温度热力图**（蓝色→橙色，默认关闭）— 每格精确温度值，可直观观察热岛效应与河流冷带
- **地表水**（蓝色，默认关闭）— 动态地表水深度（雨水+河流），透明度随深度增加
- **地下水**（靛蓝，默认关闭）— 地下水饱和度 0-1，植物消耗地下水生长
- **养分**（棕→绿，默认关闭）— 土壤养分浓度（0-10），来自尸体和植物腐败，被植物消耗

**生态位分类** — 系统根据每个 Agent 的饮食历史自动分类生态位：🌿植食（Herbivore，食物>70%）/ 🥩肉食（Carnivore，攻击>50%）/ 🦴腐食（Scavenger，尸体>50%）/ 🍽杂食（Omnivore，均衡型）。Agent 卡片上以彩色标签显示生态位（绿色/红色/黄色/灰色），每 200 tick 重新计算。生态位从底层规则涌现——Agent 不会预设角色，而是通过饮食习惯自然分化。

**追踪系统** — 右键地图上的 Agent 直接置顶追踪；双击切换追踪/取消。Agent 卡片和地图 tooltip 显示当前轮回次数（Gen N）。开启"追踪轮回"后，追踪的 Agent 死亡时自动等待其重生（同一 slot），继续追踪下一代。

**攻击伤害** — 浮动文字系统：攻击减血显示红色 "-N"，进食恢复显示绿色 "+N"，死亡显示灰色 "DEAD"，复活显示紫色 "RESPAWN"。站在食物上的 Agent 被攻击受 110% 伤害（进食脆弱性），Agent 可选择放弃食物逃跑。

**世界配置** — 创建世界时可在弹窗中配置完整模拟参数（网格尺寸、食物、温度、战斗、河流、生理），以可折叠区段展示。Dashboard 信息栏显示实时攻击率（ATK）、平均寿命、夜间死亡率等统计指标。

**昼夜与温度** — Header 显示当前世界时间（☀ HH:MM）和温度（°C）。温度影响 agent 生存压力——夜晚更危险，抱团更安全。地图可开启温度热力图，蓝色（<5°C）→绿色（15-25°C）→橙色（25-35°C）→红色（>35°C），叠加透明度 25%。

## 演化的九个阶段

这个实验有一条自然的探索路径：

**第一阶段：稳定觅食。** Agent 能否学会不饿死？这是最基本的生存能力。

**第二阶段：气味导航。** Agent 移动时留下气味（紫色），植物散发气味（绿色，强度与能量成正比）。Agent 能否学会利用两种气味梯度追踪资源？

**第三阶段：信号利用。** 广播信号有代价（3 Energy），但能传递信息。Agent 会发展出信号的语义吗？

**第四阶段：捕食与食腐。** 攻击直接扣除 Energy，尸体是可食用资源。Agent 会发展出"猎人"和"食腐者"的分化吗？

**第五阶段：策略分化。** 是否会出现稳定的生存策略类型——探索者、定居者、掠食者、食腐者、机会主义者？

**第六阶段：生态位分化。** 植物分布受水分和养分驱动。Agent 会发展出"近水觅食""高养分区定居"等策略分化吗？

**第七阶段：水源探索。** Agent 能否学会追踪河流声音找到水源？会不会围绕水源形成领地？

**第八阶段：共享语义。** 如果信号真的被使用了，不同 Agent 对同一信号的理解会趋同吗？

**第九阶段：昼夜适应与抱团。** 夜晚寒冷增加生存压力。Agent 会学会在夜晚主动靠近其他 Agent 抱团取暖吗？会演化出昼夜活动节律吗——白天觅食、夜晚聚集？

## 关于轮回

Agent 会死。但死亡不是终点。

每个 Agent 死亡后约 5 秒，在同一槽位重生——带着最新的共享策略（网络权重），但清空短期记忆（GRU 隐藏状态）。这是一种数字版的轮回：个体经验随死亡消逝，但策略智慧在种群中持续累积。

16 次轮回计为一"代"，每 8 代保存最优策略权重。世界永不停止，演化在出生与死亡的循环中持续进行。

我们观察的是策略的演化轨迹。一代又一代之后，觅食效率会不会提高？攻击频率会不会变化？信号使用率会不会上升？这些宏观趋势比任何单个 Agent 的行为都更有意义。

## 核心假设

> 复杂行为不需要被设定。
> 
> 当个体拥有生存压力、有限资源、局部感知和行动能力时，合作、竞争、交流乃至社会结构，都可能作为生存策略从生态系统中自然涌现。

如果这个假设成立，ZHI 将展示一条通往人工智能的新路径——不是通过更大的模型、更多的数据、更精巧的对齐技术，而是通过构建足够丰富的环境，让智能自己长出来。

如果这个假设不成立，ZHI 同样有价值——它会告诉我们涌现的边界在哪里，哪些行为是"自然的"，哪些行为需要程序员的引导。

无论哪种结果，都是答案。
