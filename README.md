# ZHI（栀）— Zero-Hypothesis Intelligence Ecosystem

## 我们在问什么

一个根本性的问题：

> 如果没有任何预设的社会规则，没有"合作"的指令，没有"语言"的定义，仅仅给定生存压力、有限资源、局部感知和行动能力——复杂行为能否从零开始涌现？

ZHI 是一个**最小数字生态系统**，试图用最简单的规则观察最复杂的结果。

---

## 设计哲学：不预设

系统中不存在友方、敌方、阵营、社会关系、语言含义、合作规则。

我们只定义六件事：

- **生存** — 存在值持续衰减（0.1/tick），不进食就会死；进食失败惩罚 -0.25 HP（而非 -1.0，避免低血量策略崩溃）；HP < 20 时进食奖励翻倍（+5 而非 +3），模拟求生本能；5000 tick 后渐进衰老（+0.02/tick），6000→+0.05，7000→+0.1，8000 tick 自然死亡，防止超级个体垄断种群
- **资源** — 食物有限（普通食物 +5 HP，大食物 80 HP 需 2 人协作），大食物独食只能获得 40% 能量；尸体也是资源（20 HP，300 tick TTL），死后释放气味（0.5/tick）；食物气味 0.3/tick，大食物气味 1.0/tick
- **空间** — 64×64 网格，移动有代价；随机河流贯穿地图（浅水可通行，深水不可跨越），河流每代重新生成
- **感知** — 5×5 局部视野（25 格），没有全局地图；隐蔽者 DetectionRange=2（2 格内才可见），但隐蔽者自身视野保持 5×5（伏击者保持警觉）；水流声音传播 10 格（Manhattan 距离，逐格衰减 0.9）
- **需求** — 口渴值 100→0，自然衰减 0.01/tick，移动/攻击额外 -0.05，口渴归零后持续掉血 0.2/tick；站在水边自动喝水 +5/tick
- **行动** — 8 个离散动作：上下左右移动、进食、攻击（+0.5 Stress，Stress 持续掉血 0.1/tick）、发信号（-0.25 HP，4 种离散值 0-3，语义由 Agent 自己学习）、隐蔽（HP 衰减 -10%，至少持续 5 tick，移动/进食/攻击自动解除）

仅此而已。其余一切——如果出现了的话——都是涌现的。

信号系统是最好的例子。Agent 可以广播 0-3 四种离散信号，系统不规定"0 代表食物"或"1 代表危险"。信号的语义完全由 Agent 自己学习。如果某种含义稳定出现了，那是因为它有生存优势，而不是程序员写了一个语义表。

## 为什么是"最小"

我们刻意把系统做得尽可能小：

只有**64 个 对象**，数量少意味着每个个体的生死都影响群体，资源竞争是真实的，不存在"大数定律下的平均存活"。

**46 维观察向量**，Agent 看到的是：

- [0-24] 5×5 网格编码（25 格：0=空，0.2=食物，0.4=大食物，0.6=尸体，0.5=Agent，0.8=Agent+食物重叠，1.0=自身）
- [25-28] 自身状态（HP/100，Stress/5，上次动作/6，年龄/200）
- [29-32] 信号记忆（4 通道独立衰减，每 tick ×0.9）
- [33-36] 气味梯度（北/南/东/西 四方向归一化差值）
- [37-39] 局部统计（可见食物数/5，可见 Agent 数/8，当前格气味/10）
- [40] 隐蔽状态（0 或 1）
- [41-42] 朝向向量（facing_x, facing_y，单位向量：Up=(0,-1), Down=(0,1), Left=(-1,0), Right=(1,0)）
- [43] 信号新鲜度（signal_age/20，0=刚收到，1=很久以前）
- [44] 口渴值（thirst/100）
- [45] 水流声音强度（water_sound/10，归一化）

**8 个动作**，不是连续控制空间。上下左右、吃、攻击、发信号、隐蔽。复杂行为必须从这些原子操作的组合中涌现。隐蔽状态下，其他 Agent 更难发现自己（2 格内才可见），但视野保持 5×5——伏击者保持警觉。

**极简奖励**：只奖励存活和进食，不奖励攻击、不奖励合作、不奖励通信。进食失败 -0.25（轻微惩罚，鼓励尝试），饥饿时（HP<20）进食奖励 +5（而非 +3）。攻击是否有价值，合作是否有利，信号是否有用——全靠 Agent 自己发现。

**河流与口渴**：地图中有一条随机生成的河流（浅水可通行，深水不可跨越）。Agent 有口渴值，移动和攻击会加速消耗，站在水边自动喝水。口渴归零后持续掉血。水流声音传播范围 10 格，Agent 可以听到但不知道河流的具体位置——必须自己去探索。

**渐进衰老**：5000 tick 后 Agent 开始衰老（额外衰减 +0.02/tick），6000 tick 加速到 +0.05，7000 tick 达到 +0.1，8000 tick 自然死亡。这确保世代更替真实存在，防止超级个体垄断种群。

这种设计的核心假设是：**如果一个行为在最小系统中都无法涌现，那它在更大更复杂的系统中也不太可能是"真的"涌现，而更可能是人的偏见在起作用。**

## 技术与原理

底层是 PPO（近端策略优化）+ GAE（广义优势估计），Agent 的大脑是一个 GRU 循环神经网络。每个 Agent 独立学习，没有共享梯度、没有集中训练。

每一代（Generation）64 个 Agent 在同一个世界中生存，直到全部死亡。然后通过遗传算法（MAP-Elites 行为多样性网格 + 交叉变异）产生下一代。基因是神经网络的权重，适应度是存活时间和进食量。

RND（随机网络蒸馏）提供内在好奇心奖励，鼓励 Agent 探索未见过的状态，避免陷入局部最优的"原地等死"策略。

技术栈很简单：.NET + TorchSharp 做推理，React + Canvas 做可视化，WebSocket 推送实时状态。

## 观察与调试

实时仪表盘提供多个维度的观察：

**事件监视器** — 结构化的世界事件流：进食、攻击、死亡、繁殖、隐蔽进入/退出、信号广播。每条事件带有 tick 时间戳和具体数值。能量来源占比（食物 / 大食物 / 尸体）显示在事件流顶部。

**显示切换** — 可以叠加多种可视化层：

- 气味网格（Agent 气味 / 食物气味）
- Agent 朝向箭头
- 视野扇形区域

**朝向与视野** — 每个 Agent 有明确的朝向（上次移动方向），作为观察向量的一部分传入神经网络。这意味着 Agent 可以学会转身、巡逻、跟踪、伏击等依赖方向感的行为。

**河流与声音** — 地图中有一条随机河流（浅水蓝色、深水深蓝），水流声音以 10 格范围传播。Agent 可以通过声音感知水源的存在，但必须自己探索才能找到河流位置。

**口渴状态** — 每个 Agent 的口渴值显示在状态面板中。口渴值低于 20 时变为青色警告色。

## 演化的七个阶段

这个实验有一条自然的探索路径：

**第一阶段：稳定觅食。** Agent 能否学会不饿死？这是最基本的生存能力。如果连这个都做不到，后面都不用谈。

**第二阶段：气味导航。** Agent 移动时留下气味痕迹，每 tick 衰减。能否学会利用气味梯度追踪食物、避开空旷区域？

**第三阶段：信号利用。** 广播信号有代价（-0.25 HP），但能传递信息。Agent 会发展出信号的语义吗？还是会完全忽略这个能力？

**第四阶段：捕食与食腐。** 攻击不直接给奖励，只增加目标的 Stress（持续掉血）。尸体是可食用的资源。Agent 会发展出"猎人"和"食腐者"的分化吗？

**第五阶段：策略分化。** 是否会出现稳定的生存策略类型——探索者、定居者、掠食者、食腐者、机会主义者？MAP-Elites 算法专门在行为多样性维度上搜索，为这种分化提供演化压力。

**第六阶段：原始合作。** 大食物需要多个 Agent 同时进食才能高效获取。这个机制本身不鼓励合作——它只是让合作成为一种可能的策略。合作会涌现吗？

**第七阶段：水源探索。** 河流声音传播 10 格，但 Agent 不知道河流的位置。它们能否学会追踪声音梯度找到水源？会不会围绕水源形成领地或迁徙路线？

**第八阶段：共享语义。** 如果信号真的被使用了，不同 Agent 对同一信号的理解会趋同吗？会不会出现类似"语言"的东西？

## 关于轮回

Agent 会死。全部会死。

但死亡不是终点。每一代的死亡触发遗传选择——表现好的 Agent 的神经网络权重被保留、交叉、变异，成为下一代的起点。这是一种数字版的轮回：个体消亡，但策略信息在群体层面延续。

我们观察的是策略的演化轨迹。一代又一代之后，觅食效率会不会提高？攻击频率会不会变化？信号使用率会不会上升？这些宏观趋势比任何单个 Agent 的行为都更有意义。

---

## 核心假设

> 复杂行为不需要被设定。
> 
> 当个体拥有生存压力、有限资源、局部感知和行动能力时，合作、竞争、交流乃至社会结构，都可能作为生存策略从生态系统中自然涌现。

如果这个假设成立，ZHI 将展示一条通往人工智能的新路径——不是通过更大的模型、更多的数据、更精巧的对齐技术，而是通过构建足够丰富的环境，让智能自己长出来。

如果这个假设不成立，ZHI 同样有价值——它会告诉我们涌现的边界在哪里，哪些行为是"自然的"，哪些行为需要程序员的引导。

无论哪种结果，都是答案。
