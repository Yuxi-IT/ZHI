# ZHI（栀）— Zero-Hypothesis Intelligence Ecosystem

## 我们在问什么

一个根本性的问题：

> 如果没有任何预设的社会规则，没有"合作"的指令，没有"语言"的定义，仅仅给定生存压力、有限资源、局部感知和行动能力——复杂行为能否从零开始涌现？

ZHI 是一个**最小数字生态系统**，试图用最简单的规则观察最复杂的结果。

---

## 设计哲学：不预设

系统中不存在友方、敌方、阵营、社会关系、语言含义、合作规则。

我们只定义七件事：

- **生存** — HP 持续衰减（0.1/tick），叠加连续生理惩罚（饥饿 60 起线性递增，口渴 70 起线性递增）；攻击直接扣除 20 HP（受血量比例缩放 0.2x–1.5x，实际伤害 4–30，冷却 4 tick）；饱食且补水时 HP 缓慢恢复（+0.2/tick）；5000 tick 后渐进衰老
- **资源** — 食物上限 100 个，每 10 tick 补充一个（不生成在河道、不与已有食物重叠）；普通食物 15 能量（每 tick 提取 1.0，约 15 tick 耗尽，TTL 200），大食物 80 能量（每 tick 提取 2.0，多人可同时进食竞争，TTL 400）；尸体 20 能量（每 tick 提取 1.0，约 20 tick 耗尽，TTL 300）；食物被吃后颜色逐渐变浅（能量比例可视化）
- **空间** — 64×64 网格，移动有代价；随机河流贯穿地图（5 格宽，浅水可通行，深水不可跨越），河流带有随机浅滩（可穿越的渡口），每代重新生成
- **环境** — 24 小时昼夜循环（12 现实分钟 = 1 天）；温度正弦变化（5°C 黎明 ~ 35°C 午后）；Agent 拥有体温属性（惯性追踪环境温度），散发热量（自身格 +2°C × HP比例，周围 8 格 +1°C × HP比例），受伤时体热下降；河流冷却周围（深水 -5°C，衰减 2 格）；低温（体温<15°C）触发额外 HP 衰减，抱团取暖可减缓体温下降；高温（体温>30°C）加速口渴；地图热力图叠加 + 每格温度 tooltip
- **感知** — 7×7 方向性视野（agent 面朝方向可见锥形区域，背后不可见）；没有全局地图；水流声音传播 10 格（Manhattan 距离，逐格衰减 0.9）
- **需求** — 双轴生理系统独立运作：饥饿 100→0（衰减 0.05/tick，约 2000 tick 耗尽），口渴 100→0（衰减 0.1/tick，约 1000 tick 耗尽）；惩罚为连续曲线——饥饿低于 60 即开始缓慢掉血（最高 0.25/tick），口渴低于 70 即开始掉血（最高 0.8/tick）；高温口渴加速最高 1.5x；喝水 +40 口渴
- **行动** — 10 个离散动作：上下左右移动、进食（每 tick 提取食物能量，多 agent 可同时抢食同一食物）、攻击（冷却 4 tick，+0.5 Stress）、发信号（-0.25 HP，4 种离散值 0-3，语义由 Agent 自己学习）、饮水（需靠近水源，+40 口渴）、推挤（将前方食物/尸体推一格，消耗 10 力气，失败扣 5）、地形改造（循环切换当前格地形：平地→坑→土包→平地，消耗 20 力气）

仅此而已。其余一切——如果出现了的话——都是涌现的。

信号系统是最好的例子。Agent 可以广播 0-3 四种离散信号，系统不规定"0 代表食物"或"1 代表危险"。信号的语义完全由 Agent 自己学习。如果某种含义稳定出现了，那是因为它有生存优势，而不是程序员写了一个语义表。

## 为什么是"最小"

我们刻意把系统做得尽可能小：

只有**64 个 Agent**（种群上限，可配置），数量少意味着每个个体的生死都影响群体，资源竞争是真实的，不存在"大数定律下的平均存活"。

**334 维观察向量**，Agent 看到的是：

- [0-293] 7×7 方向性网格编码（49 格 × 6 通道：普通食物/大食物/尸体/其他 Agent/自身/地形），可见区域由 Agent 朝向决定（面朝方向锥形可见，背后不可见）。通过 CNN 提取空间特征（纯卷积，无 Pooling：6ch→16@5×5→32@3×3），压缩为 288 维特征向量
- [294-298] 自身状态（HP/100，Stress/5，上次动作/10，年龄/200，力气/100）
- [299-302] 信号记忆（4 通道独立衰减，每 tick ×0.9）
- [303-306] agent 气味梯度（N/S/E/W 方向相邻格差值）
- [307-309] 局部统计（可见食物数/5，可见 Agent 数/8，当前格气味/10）
- [310-311] 朝向向量（facing_x, facing_y）
- [312] 信号新鲜度（signal_age/20）
- [313] 饥饿值（hunger/100）
- [314] 口渴值（thirst/100）
- [315] 水流声音强度（water_sound/10）
- [316] 进食状态（is_eating，1=正在进食）
- [317] 静止状态（is_stationary，1=静止休憩）
- [318-333] 信号场梯度（4 通道 × 4 方向 N/S/E/W）

**CNN + GRU 混合架构**：7×7×6 方向性网格先经两层纯卷积提取空间特征（6ch→Conv(k=3,p=0)→5×5×16→ReLU→Conv(k=3,p=0)→3×3×32→ReLU），展平为 288 维；再与其余 40 维非网格观测拼接为 328 维向量，送入 GRU(328→128)→FC(128→64)→Actor(10)/Signal(4)/Critic(1) 三头。

**10 个动作**，不是连续控制空间。上下左右、吃、攻击、发信号、饮水、推挤、地形改造。复杂行为必须从这些原子操作的组合中涌现。推挤允许 agent 将前方食物或尸体推开一格；地形改造循环切换当前格地形（平地→坑→土包→平地），为 agent 提供改造环境的能力。

**极简奖励**：只奖励存活和进食，不奖励攻击、不奖励合作、不奖励通信。攻击是否有价值，合作是否有利，信号是否有用——全靠 Agent 自己发现。

**双轴生理系统**：饥饿和口渴独立运作，各有自己的衰减速率。惩罚采用连续曲线而非硬阈值——饥饿低于 60 即开始线性递增掉血（最高 0.25/tick），口渴低于 70 即开始线性递增掉血（最高 0.8/tick）。这确保 PPO 能学习到渐进规避策略。

**昼夜与温度**：24 小时昼夜循环（12 现实分钟 = 1 天 = 3600 tick）。温度正弦变化——黎明 04:00 最冷 5°C，午后 14:00 最热 35°C。夜晚寒冷（<15°C）触发额外 HP 衰减（最高 +0.15/tick），但 agent 周围 2 格内有其他 agent 时可抱团取暖（每个邻居等效 +3°C 体感），3 个邻居基本抵消最冷夜的惩罚。白天高温（>30°C）加速口渴衰减（最高 1.5x），推动 agent 靠近水源。**Agent 体热**：每个存活 agent 散发体温（自身格 +2°C，周围 8 格各 +1°C），热量按 HP 比例缩放——受伤 agent 体热降低，濒死 agent 几乎不产热。聚集时局部温度显著升高——3 个满血 agent 抱团可使核心格升温约 6°C。**河流冷却**：深水格对周围 2 格范围降温 5°C（浅水半效），按 Chebyshev 距离线性衰减，形成沿河低温带。合作不预设——抱团是自利的生存策略。**Agent 体温**：每个 Agent 拥有独立体温属性，以 0.05/tick 速率向环境温度渐变（约 60 tick 达到平衡）。体感温度效应（冷/热伤害）基于体温而非环境温度——这意味着从寒冷区域移动到温暖区域需要时间恢复。Agent 卡片显示实时体温。Dashboard 地图提供温度热力图叠加（蓝色→绿色→橙色→红色），开启后每格 hover 显示精确温度值，可直观观察热岛效应与河流冷带。

**河流与浅滩**：5 格宽的随机河流，中心 1 格深水（不可通行），两侧浅水（可通行）。随机出现浅滩（深水变浅水），成为天然渡口。agent 需要找到浅滩才能过河，创造领土边界和交通要道。

**进食切换**：Eat 动作不再是瞬时操作，而是切换进食状态。进入进食状态后每 tick 自动从食物中提取能量，执行任何非 Eat 动作（移动/攻击/信号/饮水）即退出。进食中的 agent 受到攻击时承受 110% 伤害（进食脆弱性），必须权衡安全与能量获取。

**食物竞争**：多个 agent 同时进食同一食物时，每 tick 提取效率按 1/√n 递减（如 2 人各得 71%，3 人各得 58%），创造抢食压力。

**尸体污染**：尸体周围 2 格范围内 agent 遭受额外 HP 衰减（最高 0.06/tick 在尸体格），防止 agent 围绕尸体集群，推动探索。

**攻击冷却**：每次攻击后 4 tick 内不可再次攻击，防止连续秒杀。攻击伤害受血量比例缩放（HP 越低伤害越低，HP 越高伤害越高），鼓励健康管理而非无脑攻击。

**渐进衰老**：5000 tick 后 Agent 开始衰老（额外衰减 +0.02/tick），6000 tick 加速到 +0.05，7000 tick 达到 +0.1，8000 tick 自然死亡。这确保世代更替真实存在，防止超级个体垄断种群。

**力气与静止（Stamina & Stationary）**：Agent 拥有力气属性（0-100），移动/攻击/推挤/地形改造消耗力气，但饱食（饥饿>60）且饮水充足（口渴>70）且血量健康（>30）时每 tick 恢复 1.0 力气。连续 5 tick 不移动（允许原地进食/饮水/攻击）则进入静止状态——额外恢复速度 ×2，HP 缓慢恢复（+0.1/tick），自身格和周围格产热增加（+3°C/+2°C）。但静止时受攻击伤害 ×1.2（易伤），视野缩小为 3×3。当力气低于 10 时移动有 50% 概率失败，推挤和地形改造不可用。

**地形改造（Terraform）**：Agent 可循环切换当前格地形——平地(0)→坑(1)→土包(2)→平地(0)。坑：额外移动耗力 +2.0，视野急剧缩窄至 3×3 无方向性圆形（如"井底之蛙"），坑内攻击伤害降至 80%（低地劣势），但夜间寒冷伤害减免 30%（仅限孤立坑，连片坑失效）。土包：不可通行，阻挡视野，高地攻击伤害提升至 110%（战术高地）。不能在有任何 Agent 的格子创建土包。每次地形改造消耗 20 力气。**水利工程**：若当前格为被水淹没的坑（动态浅水），Terraform 可填水造坝——将水域变为土包（堤坝），触发 dam_built 事件。不可在河流（原生水域）上执行 Terraform。

**地形物理（风化与水漫）**：Pit 和 Mound 拥有 2000 tick 生命周期（TTL），过期自动风化回平地。靠近河流的 Pit 每 tick 可能被水淹没变成动态浅水——一旦与河流连通即为永久水域（不再蒸发干涸），形成不可逆的环境改变。Mound（土包/堤坝）可阻挡水流蔓延，Agent 可通过填水造坝主动塑造水文边界。这套机制将地形交互从"营地建设"推向"水利工程与领地防御"的博弈层面。

**推挤（Push）**：Agent 可将面前的食物或尸体推挤一格。验证目标格合法（边界内、非深水、非土包、无 Agent 且无其他实体）。成功消耗 10 力气，失败消耗 5 力气。这提供了资源操控和空间竞争的新维度。

这种设计的核心假设是：**如果一个行为在最小系统中都无法涌现，那它在更大更复杂的系统中也不太可能是"真的"涌现，而更可能是人的偏见在起作用。**

## 技术与原理

底层是 PPO（近端策略优化）+ GAE（广义优势估计），Agent 的大脑是一个 CNN+GRU 混合网络。CNN 处理 7×7×5 方向性网格提取空间特征，GRU 融合空间特征与非网格状态进行时序决策。所有 Agent 共享一个 BatchBrain（GPU 批量推理），PPO 在线持续训练。每次 PPO 更新后清空 GRU 隐藏状态，确保策略更新后不携带旧策略的时序偏差。

世界**永不重置**——采用个体轮回机制：每个 Agent 死亡后经过 5 秒延迟（25 tick）自动重生，在新位置重新开始，继承共享 PPO 策略的经验（GRU 隐藏状态清零，但网络权重持续进化）。每 16 次轮回计为一代（Generation），每 8 代保存最佳权重至数据库。

遗传算法不再用于整代替换，而是用于繁殖：存活时间长、HP 高的 Agent 可以通过无性繁殖产生后代（权重变异），后代加入种群（受种群上限约束）。

技术栈很简单：.NET + TorchSharp 做推理与训练，React + Canvas 做可视化，WebSocket 推送实时状态。

## 观察与调试

实时仪表盘提供多个维度的观察：

**顶栏** — 显示游戏时钟（☀/🌙 图标）、温度（彩色标注：白<5°C→蓝→绿→橙→红>35°C）、世代、死亡数、存活数、在线状态。DB 统计区显示平均寿命、夜间死亡率、人均攻击/进食/信号次数。

**事件监视器** — 结构化的世界事件流：进食、攻击、死亡、繁殖、复活、信号广播、推挤、地形改造、淹没、风化、填水造坝。可按事件类型过滤，支持清除和自动滚动。

**显示切换** — 可以叠加多种可视化层（默认全部开启：气味-栀、气味-食、方向、视野）：

- **气味-栀**（紫色）— Agent 移动时留下的气味痕迹，每 tick 衰减和扩散。反映 Agent 活动热点区域
- **气味-食**（绿色）— 食物和尸体散发的气味，独立于 Agent 气味扩散。大食物（2×2）四个格子各自独立发射气味，形成更强的气味云。帮助定位资源密集区
- **朝向箭头** — Agent 的朝向方向，决定视野方向
- **视野区域** — 以 Agent 朝向为基础的 7×7 锥形可见区域（静止时缩小为 3×3 圆形）
- **信号场**（黄色）— 空间中残留的信号强度（4 通道最大值），反映通信热点
- **地形**（棕色/蓝色，默认关闭）— 坑（深褐色）、土包（浅褐色）、水坑（浅蓝），hover 显示 TTL 倒计时；TTL<200 时出现风化裂纹
- **水流**（天蓝色箭头，默认关闭）— 8 方向箭头显示河流流向，仅在缩放足够大时可见

**追踪系统** — 右键地图上的 Agent 直接置顶追踪；双击切换追踪/取消。Agent 卡片和地图 tooltip 显示当前轮回次数（Gen N）。开启"追踪轮回"后，追踪的 Agent 死亡时自动等待其重生（同一 slot），继续追踪下一代。

**攻击伤害** — 浮动文字系统：攻击减血显示红色 "-N"，进食恢复显示绿色 "+N"，死亡显示灰色 "DEAD"，复活显示紫色 "RESPAWN"。站在食物上的 Agent 被攻击受 110% 伤害（进食脆弱性），Agent 可选择放弃食物逃跑。

**Settings 面板** — Dashboard 底部 Settings 标签页可在线编辑 config.json 世界参数（网格尺寸、食物、温度、战斗、河流、生理等），支持保存到磁盘和保存并重启世界。

**昼夜与温度** — Header 显示当前游戏时间（☀ HH:MM）和温度（°C）。温度影响 agent 生存压力——夜晚更危险，抱团更安全。地图可开启温度热力图，蓝色（<5°C）→绿色（15-25°C）→橙色（25-35°C）→红色（>35°C），叠加透明度 25%。

## 演化的九个阶段

这个实验有一条自然的探索路径：

**第一阶段：稳定觅食。** Agent 能否学会不饿死？这是最基本的生存能力。

**第二阶段：气味导航。** Agent 移动时留下气味（紫色），食物散发气味（绿色）。Agent 能否学会利用两种气味梯度追踪资源？

**第三阶段：信号利用。** 广播信号有代价（-0.25 HP），但能传递信息。Agent 会发展出信号的语义吗？

**第四阶段：捕食与食腐。** 攻击直接扣血（受 HP 比例缩放），尸体是可食用资源。Agent 会发展出"猎人"和"食腐者"的分化吗？

**第五阶段：策略分化。** 是否会出现稳定的生存策略类型——探索者、定居者、掠食者、食腐者、机会主义者？

**第六阶段：原始合作。** 大食物每 tick 提取 2.0 能量，但 80 总能量足够多人分食。单吃慢、人多抢得快——合作会涌现还是纯竞争？

**第七阶段：水源探索。** Agent 能否学会追踪河流声音找到水源？会不会围绕水源形成领地？

**第八阶段：共享语义。** 如果信号真的被使用了，不同 Agent 对同一信号的理解会趋同吗？

**第九阶段：昼夜适应与抱团。** 夜晚寒冷增加生存压力。Agent 会学会在夜晚主动靠近其他 Agent 抱团取暖吗？会演化出昼夜活动节律吗——白天觅食、夜晚聚集？

## 关于轮回

Agent 会死。但死亡不是终点。

每个 Agent 死亡后约 5 秒，在同一槽位重生——带着最新的共享策略（网络权重），但清空短期记忆（GRU 隐藏状态）。这是一种数字版的轮回：个体经验随死亡消逝，但策略智慧在种群中持续累积。

16 次轮回计为一"代"，每 8 代保存最优策略权重。世界永不停止，演化在出生与死亡的循环中持续进行。

我们观察的是策略的演化轨迹。一代又一代之后，觅食效率会不会提高？攻击频率会不会变化？信号使用率会不会上升？这些宏观趋势比任何单个 Agent 的行为都更有意义。

## 核心假设

> 复杂行为不需要被设定。
> 
> 当个体拥有生存压力、有限资源、局部感知和行动能力时，合作、竞争、交流乃至社会结构，都可能作为生存策略从生态系统中自然涌现。

如果这个假设成立，ZHI 将展示一条通往人工智能的新路径——不是通过更大的模型、更多的数据、更精巧的对齐技术，而是通过构建足够丰富的环境，让智能自己长出来。

如果这个假设不成立，ZHI 同样有价值——它会告诉我们涌现的边界在哪里，哪些行为是"自然的"，哪些行为需要程序员的引导。

无论哪种结果，都是答案。
